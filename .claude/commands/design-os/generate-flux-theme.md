# Generate Flux Theme

You are helping the user generate a FluxUI-compatible theme CSS file from their Design OS color tokens.

## Step 1: Check Prerequisites

Verify that design tokens exist:

Read `/product/design-system/colors.json`

If it doesn't exist:

"I don't see design tokens defined yet. Please run `/design-tokens` first to choose your color palette, then come back to generate the Flux theme."

Stop here if the file is missing.

## Step 2: Read the Color Tokens

Parse the colors.json file to get:
- `primary` — maps to Flux accent
- `secondary` — maps to custom secondary variables
- `neutral` — maps to Flux zinc reassignment

## Step 3: Generate the Flux Theme CSS

Create the file at `product/design-system/flux-theme.css` with the following structure.

**Important:** Replace `[primary]`, `[secondary]`, and `[neutral]` with the actual color names from colors.json.

```css
/*
 * FluxUI Theme — Generated from Design OS
 *
 * Primary:   [primary]   → accent
 * Secondary: [secondary] → secondary (custom, not native to Flux)
 * Neutral:   [neutral]   → zinc base
 *
 * Generated by Design OS: https://github.com/buildermethods/design-os
 */

@theme {
  /* =========================================================================
   * Base Colors (zinc → [neutral])
   * FluxUI uses zinc as its base gray. We remap all 11 shades.
   * ========================================================================= */
  --color-zinc-50: var(--color-[neutral]-50);
  --color-zinc-100: var(--color-[neutral]-100);
  --color-zinc-200: var(--color-[neutral]-200);
  --color-zinc-300: var(--color-[neutral]-300);
  --color-zinc-400: var(--color-[neutral]-400);
  --color-zinc-500: var(--color-[neutral]-500);
  --color-zinc-600: var(--color-[neutral]-600);
  --color-zinc-700: var(--color-[neutral]-700);
  --color-zinc-800: var(--color-[neutral]-800);
  --color-zinc-900: var(--color-[neutral]-900);
  --color-zinc-950: var(--color-[neutral]-950);

  /* =========================================================================
   * Accent Colors (primary → [primary])
   * Used by Flux buttons, links, navbars, tabs, etc.
   * ========================================================================= */
  --color-accent: var(--color-[primary]-500);
  --color-accent-content: var(--color-[primary]-600);
  --color-accent-foreground: var(--color-white);

  /* =========================================================================
   * Secondary Colors (custom — not native to Flux)
   * Available for custom components: bg-secondary, text-secondary-foreground
   * ========================================================================= */
  --color-secondary: var(--color-[secondary]-500);
  --color-secondary-content: var(--color-[secondary]-600);
  --color-secondary-foreground: var(--color-white);
}

@layer theme {
  /* =========================================================================
   * Dark Mode Overrides
   * Lighter accent shades for better contrast on dark backgrounds
   * ========================================================================= */
  .dark {
    /* Accent */
    --color-accent: var(--color-[primary]-400);
    --color-accent-content: var(--color-[primary]-300);
    --color-accent-foreground: var(--color-white);

    /* Secondary */
    --color-secondary: var(--color-[secondary]-400);
    --color-secondary-content: var(--color-[secondary]-300);
    --color-secondary-foreground: var(--color-white);
  }
}

/* =========================================================================
 * Utility Classes for Secondary (since Flux doesn't provide these)
 * ========================================================================= */
@layer utilities {
  .bg-secondary {
    background-color: var(--color-secondary);
  }
  .text-secondary {
    color: var(--color-secondary);
  }
  .text-secondary-content {
    color: var(--color-secondary-content);
  }
  .text-secondary-foreground {
    color: var(--color-secondary-foreground);
  }
  .border-secondary {
    border-color: var(--color-secondary);
  }
}
```

## Step 4: Confirm Completion

After creating the file, inform the user:

"I've generated your FluxUI theme at `product/design-system/flux-theme.css`.

**Color Mapping:**
- **Accent:** `[primary]` (500/400 for light/dark)
- **Secondary:** `[secondary]` (custom utilities included)
- **Base:** `[neutral]` (remapped to zinc)

**To use in your Laravel project:**

1. Copy `flux-theme.css` to your project's `resources/css/` directory

2. Import it in your `app.css` after Flux:
   ```css
   @import 'tailwindcss';
   @import '../../vendor/livewire/flux/dist/flux.css';
   @import 'flux-theme.css';
   ```

3. Flux components will automatically use your accent color. For secondary:
   ```blade
   <span class=\"bg-secondary text-secondary-foreground px-2 py-1 rounded\">
       Tag
   </span>
   ```

**Shade adjustments:** If you need different shades (e.g., 600 instead of 500 for accent), edit the generated file directly."

## Important Notes

- The generated CSS uses Tailwind CSS v4 syntax (`@theme`, `@layer theme`)
- Shade selection follows convention: 500 for light mode, 400 for dark mode
- Secondary utilities are included since Flux doesn't have native secondary support
- The file should be imported AFTER `flux.css` to override defaults
